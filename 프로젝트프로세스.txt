1. 요구사항 도출 및 계획 : 유스케이스, 요구사항 정의서, WBS
	- 요구사항 도출 -> 요구사항 정의 -> WBS 개발 계획 수립 

2. 사이트 설계 : 메뉴 설계서(사이트맵)
	- 메뉴 설계(사이트 전체 내용 도출) 
		네비게이션 설계 - 마인드맵/트리맵/카드소팅,메뉴 설계서/사이트맵(반드시 링크 내용 기입 필수) 
	- 흐름 설계 : Fluid UI 로 썸네일을 배치하여 링크로 연결한 후 전체 연결도를 완성
		예) 회원 가입 링크 -> 약관 페이지 -> 회원 가입 폼 -> 중복 아이디 체크
		-> 회원 가입 성공
		
3. 화면 설계 : 프로토타입, 스토리보드
	각 흐름에 들어 가는 화면을 설계 -> 데이터 수집, 각 화면 설계, 프로토타이핑, 화면 설계서(스토리보드)

4. 데이터베이스 설계 : 	DB ERD, 테이블 설계표
	기능 구현을 위한 RDBMS의 각 테이블, 관계 등을 설계하고, 물리적 테이블을 생성하여 더미 데이터를 추가
	
5. 클래스 설계 : 클래스 다이어그램, 시퀀스 다이어그램 또는 MVC 처리 흐름도
	다이어그램대로 해당 DTO, DAO, Connector 를 먼저 구성

6. 화면 구현(프론트 엔드) : 소스 코드, 리소스
	메인 구현 -> 공통 서브 구현
	view(.jsp) 페이지를 구성, 해당 링크를 모두 기입, 리소스를 모두 표시될 수 있도록 연결

7. 기능 구현(백엔드 - 서버프로그램 구현)
	단위(기능) 테스트 : DBTest, ListTest, DTOTest, CryptoTest~
	단위 기능 구현 : 해당 페이지 요청시에 필요한 Controller를 서블릿으로 구성하되, 각 기능들을 MVC 패턴에 맞게
	요청 -> Controller -> Service -> DAO -> RDBMS -> DAO -> Service -> Controller -> View  순으로 
	요청 데이터를 전달하고, 반환 받은 데이터가 표시될 수 있도록 구성할 것. 

※ 수행된 내용은 메뉴설계서, 흐름 설계도, 프로토타입과 시연장면 녹화본, 스토리보드 등을 포트폴리오 형식으로 제출할 것.
